insa毕业生找工_2020

Posted on 2020-09-24

2020是个不平常的一年。我计划是insa毕业之后去读北美的ms项目，中途因为疫情还有其他种种原因，在找工和继续读书两者之间疯狂反复横跳。不过还是非常用心的尝试找工这件事。想着学弟学妹也许也会经历这样的过程，就决定写一写在欧洲找工的心路历程。

因为是从informatique系毕业，所以个人方向是纯技术方向，让我去尝试一些需要social能力的岗位我觉得也不太行。教育背景很偏dev，自己也比较喜欢。不过后来因为一些原因，走了和data相关的方向。

我会以找工目的地为分类，先开始介绍一些我关注过的公司。 然后是一些找工心得。最后是我想谈的一些感想。

一些公司

1. 法国的一些公司。

insa毕业找法国公司的优势在于法国的公司好歹知道insa。 知道你苦逼读了5年工程师，也不容易（笑）。 然后如果是insa lyon的话，工资好歹在应届生第一梯队（这个我也不太确定，只能从自己有限的已知信息推测）。当然，法国稍小一些的公司本身工资差距就很小，也没什么计较的必要，也就是一些大的企业，比如几家银行，金融公司，对不同学校给的offer能看出一些差距来。

1.1 法国找工我认为data方向这些公司值得一投：

第0顺位： amazon， Google， airbnb 在法国的分部。 这些公司不用说了，工资高，title好。但是招人也很少，招人不规律，岗位大多是缺啥招啥。应届生更难。随缘吧，常去官网看看。
我认识有学长在巴黎的亚麻实习过，谷歌没了解过有谁去， airbnb在法国招data scientist，但是ds太需要数学统计背景，insa if 的profil肯定不够。在领英上看到的基本上是ens的这个级别的学生去。 不过今年疫情airbnb大裁员，最近一两年都不用想了。fb在巴黎有ai reserach center， 但是应该需要读博以及connection才能去实习。 工程师和这种太research方向无缘。

sg, bnp, axa, deloitte等一些金融机构。我放在最前面是和我方向关系比较大。金融机构需要data方向的人比较多，这个和公司业务相关，本身大型互联网公司也需要data的人，但是毕竟欧洲是互联网荒漠。。所以。。。其次还有原因是公司体量大，在法国reputation也比较好，工资还行。

然后是一些本地互联网企业，datadog，criteo这两个是经常拿到一起说的，主要原因是都在美股上市。也算是法国唯二能说一说的国际化互联网公司了？。。 但是不得不说criteo股价还有公司前景都不太被看好，所以招人也就非常不aggressive，尝试投过，石沉大海。 datadog投实习的时候尝试过，邀请了面试，但是当时已经确定要去sg实习就没有再面下去。现在想想有点可惜，应该试试看的。 datadog对new grad毫无工作经验的毕业生不友好，不知道是不是不sponsor工签的原因，总之我投full time的时候，没有给我面试。可能通过实习转正可以以new grad的身份加入（这个不确定）

再之后是咨询公司了，说实话咨询公司我分不出啥顺序，因为咨询公司本身是要把你派去别的公司干活的，所以更重要的是你被派去哪里。当然投之前要看好咨询公司都和哪些公司有合作，如果没有你想去的公司，那还是算了吧。

2. 法国地区之外欧洲的公司。

除法国之外的地区，对insa的学历认可就低一些了，不过很多也值得一试。
如果你对互联网行业比较了解的话，应该知道互联网巨头主要都在美国，其次是中国。那欧洲没有咋办呢– 幸运的是欧洲还有一些美国互联网的分部。 主要分布在：德国，都柏林，荷兰，英国，瑞士。
德国最主要是亚麻还有fb，亚麻招人规模比较大，招聘节奏也比较规律。不方便的地方是很多岗位需要你会德语英语。。
都柏林因为税收原因基本上我们认识的所有美国大厂都在都柏林有分部。 不过据说headcount也很少。非欧洲学生需要公司sponsor工签，所以是难度高于欧洲学生的。我们这届很多欧洲学生选择去都柏林交换或者双学位也是奔着那里公司多去的。
荷兰我关注的主要是databricks, uber。 uber在阿姆斯特丹office比较大，但是因为疫情影响，最近不用想了。
英国主要是伦敦那里，headcount也不多，需要盯着官网看有没有相关的岗位招人。有在法毕业的中国学生直接去的先例，做的是ml engineer。
最后是瑞士，瑞士工签需要三年工作经验才能办。然后优先顺序是瑞士本地人大于欧洲护照大于非欧洲外国人，所以想去瑞士也不容易。优点是工资很高，和美帝有的比。

3. 国内的公司。

国内互联网发展的很快，有大批的公司在招人。 字节，bat，美团，京东，滴滴，b站都招聘大量的技术岗。如果真的想做技术，其实国内的公司也值得一去。当然工作强度和压力也和国外公司不在一个层次：）

找工心得：

1. 精通至少一个方向

这点在面非美国厂的时候尤为重要。 insa一礼拜有30+小时的课，然而你找到一份工作只需要你在一个方向精通。 精通意味着你需要会使用相关方向在工业界的技术，比如java开发需要熟悉java ee框架， 大数据需要熟悉MR， Spark， 前端需要angular， react… 这些工业界现行框架是动态的， 每年都会有新的框架面世，也有现存框架逐渐被淘汰。 简历上一定要有现行框架的经验，实习，project都行。框架原理也是肯定会出现在面试中的。也需要了解这个方向的上下游环节，和哪些别的方向交互，在公司为哪些职能服务。

2. 不同方向的岗位数量会影响你找工的难易度。

现在技术方面主要的方向有：前端开发，后端开发，数据库相关，devops，游戏方向，算法工程师（近似于data scientist或 ml engineer）， 大数据开发（data engineer, data warehouse engineer, big data engineer)，移动端开发……
最典型的例子的是国内的算法工程师方向，19年我在国内实习的时候，面算法的同学就告诉我那年和18年比，难了许多。主要原因是ai方向逐渐成型，没有那么多投资涌入，公司也就不再乐意在相关方向投入太多资金了。20年国内cv方向已经是神仙打架了。 15年前后移动端开发遍地跑，到了这两年已经很少听到人谈起安卓，ios开发了。国内的游戏市场最近发展非常迅速，知名大厂都努力在游戏领域分一杯羹，今年游戏方向招聘也挺火爆。 然而很可能在欧洲就很难加入一个知名的游戏厂。 所以选好方向也很重要。

3. 算法很重要。

不说了，请刷leetcode。

法国vs国内vs别的国外厂：

作为一个到处找工的人，我比较了不同地区的找工经历，总结出来一些经验：

1. 公司招你的前提条件是当地的工签政策允许。

一个没有工签的人再牛逼也没法工作。中国公民在国内工作公司雇佣你的成本是很低的，因为不需要给你办理工签。 在欧洲国家，公司是必须要为非欧洲护照外国人付费办理工签的，这是成本，也是风险，所以在招聘过程中公司会考虑这方面。一个很明显的例子是在投递都柏林亚麻的时候，很多外国学生面试流程停滞，但是仍然有欧洲护照的学生接到下轮面试。 这很有可能是给外国人headcount用完造成的。

近几年有很多中印做技术的直接肉身翻墙去德国公司，主要原因是德国的蓝卡政策允许做技术的获取工签。这也是我觉得如果可以毕业加入国内一些不错的公司也未尝不可，毕竟来欧洲的门是始终打开的：招聘一个欧洲毕业的学生和从国内有工作经验的人都需要公司办理工作签证，成本是一样的，那公司自然更倾向于招聘一个有经验的人。

2. 面试模式。

国内公司还有法国我面的一些公司的面试模式是大致一致的，主要考察第一算法数据结构基础，以及你对相关框架的应用。 这种面试方式算是面向岗位面试，你从一面试一开始就已经确定好了能去的team，所以公司更在乎你能否快速上手，快速出活。这点在国内面试体现的淋漓尽致：你的面试内容会和公司业务紧密相关，如果和相关team的日常工作技术点不切合，很容易挂掉。 一些美国厂则只注重算法数据结构。但是并不是只刷leetcode就足够了的，不同公司也有不同的要求：比如亚麻就是bq大厂，谷歌从不出原题，有的公司要求你一定要会多线程，ood。

3.难度。

不同的面试模式之间很难比较哪个更难，但是可以肯定的是相同岗位法国这里的难度比国内简单。 主要原因还是因为国内工程师红利，不缺人写码。 法国本来会说法语的工程师就不多，自然竞争小。
