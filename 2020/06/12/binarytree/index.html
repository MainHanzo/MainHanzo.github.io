non recursive binary tree traversal

Posted on 2020-06-12 In leetcode

The most classic questions about binary trees are about binary tree traversals.
We all know that we can traverse the binary trees recursively in preorder, inorder or postorder.
If we traverse the tree recursively, the algorithme is intuitive. What if we need to traverse it iteratively?

Most recursive questions can be solved with stack. Here, I want to illustrate a general solution to traverse the binary tree iteratively. This can be also the solution to leetcode 94, 144, 145.

The idea is to stock the nodes encoutered with a stack. However, the nodes stored in stack have two types. The first type is the node to visit, the second type is the node to print. Each node to print must be visited first. Let’s see the code of leetcode 145.
